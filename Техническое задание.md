# Техническое задание: FarmQuest

## 1. Общее описание

FarmQuest — веб-игра «ферма», в которой игрок управляет участком земли, сажает и собирает урожай, ухаживает за животными и выполняет простые мини-игры (например, рыбалка или «Поймай морковки»), чтобы получать внутриигровую валюту и развивать ферму.

Цель проекта — реализовать учебное веб-приложение с использованием системы контроля версий Git и организации командной разработки через GitHub.

## 2. Тип проекта

Тип проекта: веб-приложение с клиентской частью в браузере и серверной частью на Node.js.

## 3. Инструменты и стек технологий

- Язык разметки и стилизации: HTML, CSS  
- Язык программирования на клиенте: JavaScript  
- Серверная платформа: Node.js  
- Веб-фреймворк: Express  
- База данных: SQLite  
- Система контроля версий: Git  
- Платформа хостинга репозитория: GitHub  

## 4. Функциональные требования (MVP)

1. Регистрация и авторизация пользователя.
2. Экран фермы с сеткой полей, где можно сажать и собирать урожай.
3. Мини-игра (минимум одна) для получения внутриигровых монет  
   – например, рыбалка или «Поймай морковки».
4. Внутренняя экономика: начисление и списание монет, покупка семян и/или улучшений.
5. Сохранение прогресса игрока в базе данных (уровень, ресурсы, состояние фермы).

## 5. Нефункциональные требования

1. Работа в актуальных версиях современных браузеров (Chrome, Firefox, Edge).  
2. Базовая адаптация интерфейса под настольные и мобильные устройства.  
3. Код проекта хранится в публичном репозитории GitHub.  
4. Разработка ведётся через задачи (Issues), ветки и Pull Request, с обязательным код-ревью.  

## 6. План спринтов и задачи

### Sprint 1 – Постановка задачи

1. Первоначальная настройка проекта (создание репозитория, базовая структура).
2. Настройка репозитория и документации (README.md, Техническое задание).
3. Создание Milestone-ов и плана спринтов, определение общего объёма задач.

### Sprint 2 – Базовый функционал

1. Реализация регистрации и авторизации пользователя.
2. Создание базовой структуры главного экрана фермы (сетка полей).
3. Настройка базы данных игроков в SQLite.
4. Реализация API для работы с профилем пользователя и прогрессом.
5. Связь клиентской части с серверной (отправка и получение данных через API).

### Sprint 3 – Игровой функционал

1. Реализация мини-игры «Рыбалка» или аналогичной по сложности.
2. Реализация мини-игры «Поймай морковки» или аналогичной по сложности.
3. Разработка системы внутриигровых монет и баланса.
4. Создание магазина семян/улучшений с использованием внутриигровой валюты.
5. Введение уровней фермы и прогресса игрока (навыки, открытие контента).

### Sprint 4 – Завершение и деплой

1. Доработка и улучшение пользовательского интерфейса (UI-полировка).
2. Реализация адаптивного дизайна для основных разрешений экранов.
3. Добавление локализации интерфейса (минимум русский, при возможности английский).
4. Написание и выполнение базовых тестов (ручных и/или автоматизированных) ключевых сценариев.
5. Подготовка и деплой проекта на выбранный хостинг, финальная проверка работоспособности.
